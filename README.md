«Лисий диспетчер» можно сразу открыть из `dist/index.html`: готовая версия собрана в один автономный HTML-файл и работает непосредственно в браузере.
Это интерактивный отчёт по сценарию тестового задания MOX для AI-first Developer.
Он собирает записи по `fox_id`, выставляет каждой лисе балл и объясняет положение лидера в рейтинге.

Для разработки понадобится Node.js версии 20 или новее.
Проект запускается командами:

```bash
npm install
npm run dev
```

Команда `npm run check` запускает ESLint, затем пять модульных тестов и production-сборку.

Для браузерной проверки нужен Python 3.11 или новее.

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements-e2e.txt
.venv/bin/playwright install chromium
```

В первом терминале нужно запустить приложение на проверяемом порту.

```bash
npm run dev -- --host 127.0.0.1 --port 4173
```

Во втором терминале запускается сценарий Playwright.

```bash
.venv/bin/python e2e/validate_app.py
```

Основная работа с данными происходит в журнале наблюдений.
В каждой записи редактируются `fox_id`, локация, окрас, уровень подозрительности, наличие добычи и время.
В журнал можно добавить строку, удалить существующую запись или вернуть первоначальные данные.
Рядом находятся четыре ползунка с весами сигналов.
При их изменении приложение без перезагрузки пересчитывает баллы, рейтинг, лидера и подробный разбор результата.
Семь этапов работы собраны в отдельной вкладке AI Worklog.

При расчёте средний уровень подозрительности каждой лисы умножается на выбранный вес.
К результату прибавляются баллы за наблюдения с добычей, повторные встречи и повторные появления в главной локации.
Бонус за повтор не начисляется ни первому наблюдению, ни первому появлению в локации.
Посмотреть саму формулу и вклад отдельных сигналов можно в интерфейсе.

Пять модульных тестов подтверждают исходного лидера и его балл, корректную смену лидера после изменения данных, агрегацию локаций, обработку пустого набора и состояние ничьей при нулевых весах.
Дополнительный сценарий Playwright выполняется в широком и мобильном представлениях.
Он редактирует исходные данные до появления другого лидера, открывает формулу и AI Worklog, добавляет новую строку и отслеживает горизонтальное переполнение, ошибки страницы и сообщения об ошибках в консоли.

Codex использовался для разбора требований, сравнения вариантов скоринга, подготовки черновиков React и TypeScript, написания тестов и ревью.
Решения по формуле, структуре интерфейса и финальной приёмке принимались после ручной проверки результата.

Логика расчёта и агрегации лежит в `src/report.ts`, интерфейс приложения в `src/App.tsx`.
В `tests/report.test.ts` расчёт проверяется через публичную функцию.
Файл `e2e/validate_app.py` отвечает за пользовательский сценарий в Chromium, а `dist/index.html` содержит автономную production-сборку.

Одна лиса определяется по стабильному `fox_id`.
Рассчитанный балл служит только для приоритизации наблюдений, это не вероятность.
Постоянного хранения нет: данные остаются в памяти браузера и после перезагрузки заменяются исходным набором.
Backend, авторизация, внешний API и постоянное хранилище отсутствуют, поскольку тестовое задание их не требовало.

Проект использует React 19, TypeScript, Vite, Vitest, ESLint и Python Playwright.
